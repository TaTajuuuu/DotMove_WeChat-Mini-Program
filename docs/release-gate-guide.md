# 发布门禁操作指南

本指南用于完成体验版测试前的三个发布门禁。所有操作先在当前开发环境完成，不要直接在生产环境试错。

## 1. 清理验收样例与停用模拟数据

代码侧已经完成：

- 回顾页不再展示“生成验收样例”。
- 小程序端和云函数端不再提供 `createDemoArchive`。
- 首页停用的 `if (false)` 模拟小组代码已移除。

云数据库中的历史验收样例不会由代码自动删除。需要在云开发控制台手动处理：

1. 打开“云开发控制台 -> 数据库 -> archiveMemberSnapshots”。
2. 查找 `groupId` 以 `demo_archive_group_` 开头的记录。
3. 逐条确认并删除这些成员快照。
4. 打开 `archiveSnapshots`。
5. 查找 `groupId` 以 `demo_archive_group_` 开头，或 `groupName` 为“月度运动验收样例”的记录。
6. 逐条确认并删除这些归档快照。
7. 重新编译，确认回顾页只显示真实归档。

不要删除真实小组产生的归档记录。

## 2. 当天修改打卡与补卡

已实现的入口：

1. 进入“打卡 -> 查看打卡记录”。
2. 点击一条当天提交的打卡或补卡记录。
3. 详情页显示“修改记录”。
4. 可修改运动数值、三环闭合、照片和备注。
5. 保存后业务状态变为 `edited`，内容审核状态重新变为 `pending`。

验收：

- 当天记录显示修改入口。
- 昨日提交的记录不显示修改入口。
- 修改后仍要求 1 至 3 张图片。
- 不能修改其他成员记录。
- 修改成功写入 `CHECKIN_EDIT` 审计日志。

## 3. 配置用户隐私保护指引

代码侧已经启用 `__usePrivacyCheck__`，选择照片前会调用隐私授权，并提供“隐私指引”入口。

在微信公众平台执行：

1. 登录微信公众平台，进入对应小程序。
2. 打开“设置 -> 服务内容声明 -> 用户隐私保护指引”。
3. 声明会处理“用户选择的照片或视频”。
4. 用途填写：用户上传运动证明照片，用于同一运动小组内的打卡展示与相互监督；内容安全审核通过前仅提交者可见。
5. 声明会使用“摄像头”，用途填写：用户现场拍摄运动证明照片并上传。
6. 声明会处理“发布内容”。
7. 用途填写：用户发布运动数据和备注，用于打卡详情展示及内容安全审核。
8. 声明会使用“剪切板”，用途填写：用户主动点击复制小组邀请码，便于发送给待加入成员。
9. 不要声明“相册（仅写入）权限”；项目不调用保存图片到系统相册的接口。
10. 说明保存位置为微信云开发云存储和云数据库。
11. 说明可见范围为同组 active 成员，且仅限内容审核通过后的记录。
12. 保存并提交隐私指引审核。

真机验收：

- 首次选择照片时出现隐私授权流程。
- 拒绝授权后不能拉起照片选择器。
- 同意后可正常选择照片。
- 点击“隐私指引”可打开平台配置的正式文本。

## 4. 创建内容审核任务集合

在“云开发控制台 -> 数据库”创建 `contentReviewTasks`，权限设为“所有用户不可读写”。

添加索引：

| 索引名 | 属性 | 字段 |
|---|---|---|
| `uniq_traceId` | 唯一 | `traceId` 升序 |
| `idx_recordId_revision_status` | 非唯一 | `recordId`、`revision`、`status` 均升序 |

在 `checkinRecords` 添加非唯一索引：

| 索引名 | 字段 |
|---|---|
| `idx_contentReviewStatus_updatedAt` | `contentReviewStatus`、`updatedAt` 均升序 |

## 5. 部署内容安全代码

1. 在微信开发者工具中右键 `cloudfunctions/yidianApi`。
2. 选择“上传并部署：云端安装依赖”。
3. 确认云函数 OpenAPI 权限包含：
   - `security.msgSecCheck`
   - `security.mediaCheckAsync`
4. 右键 `cloudfunctions/contentReviewCallback`。
5. 选择“上传并部署：云端安装依赖”。

两个云函数设置相同的环境变量：

| 环境变量 | 说明 |
|---|---|
| `CONTENT_REVIEW_CALLBACK_SECRET` | 内部调用密钥，建议随机 32 字节以上 |

`contentReviewCallback` 额外设置：

| 环境变量 | 说明 |
|---|---|
| `CONTENT_REVIEW_CALLBACK_TOKEN` | 微信消息服务器明文回调 Token，建议随机 16 至 32 字符 |

不要把这两个值写入小程序端代码或提交到 Git。

## 6. 配置异步图片审核回调

`mediaCheckAsync` 的审核结论通过消息服务器回调返回。必须完成：

1. 在云开发控制台为 `contentReviewCallback` 开通 HTTP 访问服务。
2. 复制 HTTPS 访问地址。
3. 在微信公众平台进入“开发管理 -> 开发设置 -> 消息推送”。
4. URL 填写该 HTTPS 地址。
5. Token 填写 `CONTENT_REVIEW_CALLBACK_TOKEN` 的同一值。
6. 消息加解密方式选择“明文模式”。
7. 保存时平台会发 GET 验证；函数会校验签名并返回 `echostr`。

当前回调函数只支持明文模式。不要选择兼容模式或安全模式。

## 7. 内容安全验收

### 合规内容

1. 提交带备注和 1 张普通运动照片的打卡。
2. `checkinRecords.contentReviewStatus` 应先为 `pending`。
3. `contentReviewTasks` 应出现对应 `traceId`。
4. 回调后任务变为 `passed`，记录变为 `passed`。
5. 通过前记录只在本人历史中显示“审核中”，不计入统计。
6. 通过后组员可见，且计入目标进度。

### 不合规内容

1. 使用测试环境验证不合规文本被 `msgSecCheck` 阻止写入。
2. 图片审核不通过时，记录变为 `rejected`。
3. 该记录 `photos` 变为空，云存储文件被删除。
4. 记录仍占用物理 5 次上限和补卡提交次数。
5. 记录不计入统计，不对其他成员展示。
6. 提交者在提交当天可进入详情，补充合规照片后重新送审。

### 异常处理

- `failed` 表示审核任务提交失败，不计入统计且不向组员展示。
- 使用 `traceId`、`recordId`、`contentRevision` 排查。
- 旧版本回调不会覆盖用户后来修改并重新送审的版本。

## 8. 提交体验版前检查

1. 重新编译并清除开发者工具缓存。
2. 部署两个云函数。
3. 完成新集合、索引和环境变量。
4. 用真机完成合规内容审核闭环。
5. 用两个不同微信账号确认 pending 内容不会被另一成员看到。
6. 确认隐私保护指引审核状态有效。
7. 完成后再上传体验版；未完成异步回调闭环前不要正式提审。
