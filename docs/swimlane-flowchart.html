一点运动微信打卡小程序 v1.0 泳道流程图

依据：Constitution v1.0.9、PRD v1.0.9、Spec v1.0.12 范围：创建、加入、目标、打卡、补卡、查看、统计、成员管理、归档 口径：按北京时间自然日和自然月计算

本图把端到端流程拆为 6 条泳道：创建者、成员、小程序页面、云函数/服务校验、数据状态、统计与归档。横向按业务阶段推进，纵向展示同一阶段中各角色和系统职责的交接关系。

关键规则索引

权限与可见性

  • 创建者必须同时拥有该小组 active 成员关系。
  • active 成员可查看同组成员的目标、打卡照片、备注和统计。
  • exited、removed、dissolved 后普通页面不可见，不计入当前统计。

打卡与补卡限制

  • 仅 active 小组、active 成员、已设置目标时可提交。
  • 同一成员、同一小组、同一运动日期最多 5 次物理有效记录。
  • 每个用户每自然日最多 3 次补卡，且只能补昨日或前天。

统计与归档

  • 统计使用有效目标配置和具备统计资格的 valid / edited 记录。
  • 月目标完成以所有已设置目标均达到 100% 为准。
  • 归档时冻结快照，归档后普通页面不再重算。
